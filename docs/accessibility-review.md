# Accessibility review

Implemented: landmarks and headings; skip link; semantic buttons/inputs; persistent form labels; visible focus; keyboard-openable ledger rows; arrow-key collection tabs; meaningful image alternatives; aria-live search/pricing/toast regions; inactive-view `hidden` and `aria-hidden` state; route focus management; dialog semantics; Escape/backdrop/browser-Back close; app-shell isolation; expanded-state announcements; focus trap and restoration; 44px practical action targets; no color-only price status; reduced-motion support; no hover-only or drag-only operations; camera has file/manual alternatives.

Manual verification still required on iOS VoiceOver, Android TalkBack, Windows screen reader, 200% zoom, and a physical keyboard. Production charts need visible timeframe/units and a summary table when real history exists.
